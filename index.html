Project 1 • Taskwork

A user-friendly project management tool that utilizes the scrum framework, allowing users to track project tasks, review those in flight and boost team efficiency. Taskwork was built by developers for development teams offering a project management tool with a relatively low barrier to entry and fluid UI/UX design.

Built with HTML + CSS + Node.js + Express + Postgres

Users create projects and track their overall status through sprints. Each project is linked to its own task page, where users can add tasks and advance them through different stages of completion.

Built by The Harmonies

Taskwork was a team effort built by Luke Brazil, Jai Gokhale, Kevin Jeffers and myself. The app was conceptualized and designed as part of our DigitCrafts Coding Bootcamp to showcase Fullstack Development skills.

Connect with the team on LinkedIn Luke, Jai, Kevin and Harmony!


Project Features!

User Login and Form Validation
The app allows users to create an account or sign in as guest, both methods utilize bcrypt to encrypted passwords. Form validation looks for the @ and .com in the email address field at time of login.

Create Projects and Tasks
Users have access to their unique projects and tasks, they can progress them through different stages of completion using arrows to move and delete those no longer needed.



My Contributions.

Product Manager and UI/UX Lead
As the Product Manager and Lead UI/UX Designer, it was my responsibility to design a mobile-first app that was fully responsive while keeping the project on schedule. I empowered our development team to focus on Taskwork's MVP, facilitied daily stand-ups and utlized Trello to track progress. The design builds off the Bulma Framework, with custom CSS and graphics implemented. Logo creation, graphic design and brand implementation all fell under my responsibility.



Project 2 • To-Do Today

To-Do Today provides information about local events in select cities that are happening Today! Users can browse information about weather, concerts, festivals, sporting events, breweries and more.

Built with HTML + CSS + JavaScript + JSON

To-Do Today's functionaility comes from combining five APIs: OpenBreweryBD, OpenWeatherMap, PredictHQ, TicketMaster and Zomato.

Built by Three Guys and an Aussie

To-Do Today was a team effort built by Dylan Cooper, Matthew Everett, Justin Gardner and myself. The project was completed in a five day scrum sprint.

Connect with the team on LinkedIn Dylan, Matthew, Justin and Harmony!


Project Features!

Provide Weather and Local Events
Users get access to the local weather and a list of events that are happening today. Feel free to spontaneously plan!

Select Cities
To-Do Today is currently available in Atlanta, Houston, Seattle and Tampa.

Category Selections
Select categories to add or remove events of interest, such as community events, concerts, craft breweries and cidery, expos, festivals, performing arts, restaurants and sporting events.



My Contributions.

UI/UX Lead and Brand Development
As the Lead UI/UX Designer and Front-end Developer, it was my responsibility to design a mobile-first app that was fully responsive, easy to use and functional on Chrome, iOS and Andriod. The design builds off the Bulma Framework, with custom CSS and graphics implemented. The To-Do Today brand draws inspiration and coloring from sunrise and sunset, allowing the user to see each day in a unique light. Logo creation, graphic design and brand implementation all fell under my responsibility.



Project 3 • Sean vs AI

Sean vs AI is a fully interactive puzzle game built in Python using Pygame. Playing the game as Sean, a master coder who has a date with destiny, the goal is to solve coding problems that would be familiar to an entry-level developer. Tackle questions on dictionaries, booleans, if and then statements, and for loops to defeat the Evil AI and save DigitalCrafts.

Built with Python3 + Pygame

Sean vs AI's gameplay comes from combining text based questions and answers, with reto inspired graphics that utilize Pygame for player animation and screen bliting.

Built by Witty, Thought Provoking Team Name

Sean vs AI was a team effort built by Aliyah Gamble, Jai Gokhale, Eric Schorling and myself. The game was completed in a four day sprint only 2 weeks into our web development bootcamp.

Connect with the team on LinkedIn Aliyah, Jai, Eric and Harmony!


Project Features!

Player Animation
Utilizing Pygame's built in event.get() method and multiple PNGs, the gamer is able to move around the screen and enjoy player animation.

Multiple Levels and a Hidden Final Challenge
Playing as Sean, gamers will tackle 4 levels. Answering specific coding questions and keeping track of a randomized secret code, needed to proceed from level to level. The final level shows up once all the puzzles have been solved.



My Contributions.

Player Animation and Gameboard Designer
In creating Sean vs AI, my responsibilities included designing the multiple gameboards and animating the player. My design process included sketching to get initial ideas on paper, then tools such as Canva, Bitmoji, Pygame and Python3 to create the finished design.
